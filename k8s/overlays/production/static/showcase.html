1. Document Upload

Documents are submitted via webhook with title, source, and content fields. The API validates and sanitizes all input to prevent injection attacks.

2. Text Chunking

Large documents are split into 500-character chunks with 50-character overlap. This ensures context preservation at chunk boundaries while keeping embeddings focused.

3. Embedding Generation

Each chunk is converted to a 768-dimensional vector using nomic-embed-text via Ollama. This captures semantic meaning, not just keywords.

4. Vector Indexing

Embeddings are stored in Elasticsearch with dense_vector type and cosine similarity indexing for fast approximate nearest neighbor search.

1. Query Embedding

User's natural language question is converted to the same 768-dim vector space using nomic-embed-text. This allows semantic matching.

2. Cosine Similarity

Elasticsearch computes cosine similarity between query vector and all indexed document vectors. Values range from -1 to 1, with 1 being identical.

3. Top-K Retrieval

The most similar chunks are retrieved (default: 5). Results include the original text, source metadata, and similarity score as percentage.

4. RAG Augmentation

Retrieved chunks can be injected into LLM prompts as context, enabling the model to answer questions using your specific knowledge base.

Key Technical Aspects

Local Embedding Model

768-dimensional vectors optimized for retrieval. Runs entirely on-premise with no external API calls required.

Smart Chunking

Documents split into overlapping segments to preserve context at boundaries while keeping embeddings focused and relevant.

Semantic Matching

Cosine similarity compares meaning, not keywords. Find relevant content even when exact terms don't match.

Visual Workflow Builder

No-code pipeline configuration. Easily modify processing steps, add data sources, or adjust parameters.
